◆ process_data()

void ServerSocket::process_data ( )

process the data received by the client orders. It is meant to be called only by Lwip on the receive_callback

reads all the data received by the server in the ethernet buffer, packet by packet. Then, for each packet, it processes it depending on the order id (default behavior is not process) and removes it from the buffer. This makes so the receive_callback (and thus the Socket) can only process declared orders, and ignores all other packets.
